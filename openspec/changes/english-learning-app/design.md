# 课后英语复习 App — 设计说明书

## 1. 产品概述

一款面向 **7-10 岁儿童家庭**的课后英语复习工具。家长在课后通过拍照录入当天学习的单词和句子，孩子可以反复听发音、跟读录音、对比练习。

**核心约束：**
- 仅在家庭内部使用，不发布到 App Store / Google Play
- 零成本（无需服务器、无需付费开发者账号）
- 所有数据本地存储，不上传任何云端
- 同时支持 iPhone 和 Android

**产品形态：** PWA（渐进式 Web App）
- 通过 Safari（iOS）或 Chrome（Android）访问一次即可"添加到主屏幕"
- 全屏运行，体验接近原生 App
- 首次加载后完全离线使用

### 3.4 M3 — 发音播放

**方案：Web Speech API (SpeechSynthesis)**

```javascript
// 播放标准发音
function speak(text: string) {
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.lang = 'en-US';
  utterance.rate = 0.8;    // 稍慢，适合孩子
  utterance.pitch = 1.0;
  utterance.volume = 1.0;
  speechSynthesis.speak(utterance);
}

// 选择最佳语音
function getBestVoice() {
  const voices = speechSynthesis.getVoices();
  // 优先选择英语母语者声音
  return voices.find(v =>
    v.lang.startsWith('en-US') && v.localService
  ) || voices.find(v => v.lang.startsWith('en'));
}
```

**注意事项：**
- iOS Safari 的 `speechSynthesis.getVoices()` 需要在用户手势事件后调用
- 首次调用 `speechSynthesis.speak()` 会弹出"是否允许语音"提示（仅一次）
- 不同手机系统的声音效果不同，无法统一

**录音实现：**

```javascript
// 录音模块
class RecordingManager {
  private mediaRecorder: MediaRecorder | null = null;
  private audioChunks: Blob[] = [];

  async start() {
    const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
    this.mediaRecorder = new MediaRecorder(stream);
    this.audioChunks = [];

    this.mediaRecorder.ondataavailable = (e) => {
      this.audioChunks.push(e.data);
    };

    this.mediaRecorder.start();
  }

  stop(): Blob {
    return new Promise((resolve) => {
      this.mediaRecorder!.onstop = () => {
        const blob = new Blob(this.audioChunks, { type: 'audio/webm' });
        this.mediaRecorder!.stream.getTracks().forEach(t => t.stop());
        resolve(blob);
      };
      this.mediaRecorder!.stop();
    });
  }
}
```

**格式说明：**
- iOS Safari 录制格式为 `audio/mp4` 或 `audio/webm`（取决于版本）
- Android Chrome 通常为 `audio/webm;codecs=opus`
- 统一使用 `Blob` 存储，IndexedDB 支持直接存储 Blob

  ┌──────────────────────────────────┐
  │  标准音  ▂▄▆█▇▆▄▂▁               │
  │  你的音  ▂▃▅▇▆▅▃▁                │

**实现思路：**
1. 使用 `AudioContext` 解析音频数据
2. 提取波形峰值数据（`AnalyserNode.getByteTimeDomainData`）
3. 在 Canvas 上绘制波形
4. 简单的匹配度计算：对比两段音频的能量分布相似度

> **注意：** 匹配度仅用于趣味性，不是精确的发音评分。

### 3.7 M6 — 复习模式

```
  复习模式流程：
  ════════════

  ┌────────┐    ┌────────┐    ┌────────┐    ┌────────┐
  │ 随机取词 │───▶│ 听音辨词 │───▶│ 跟读练习 │───▶│ 标记结果 │
  │        │    │        │    │        │    │        │
  │ 从词库  │    │ 播放    │    │ 录音    │    │ 对/错   │
  │ 抽取 10 │    │ 标准音  │    │ 对比    │    │ 记录    │
  │ 个单词  │    │ 孩子选  │    │ 波形    │    │ 熟练度  │
  └────────┘    └────────┘    └────────┘    └────────┘
```

**熟练度算法（简单版）：**

```
  每次复习后更新 masteryLevel (0-5):
  - 标记"对" → masteryLevel + 1 (上限 5)
  - 标记"错" → masteryLevel - 1 (下限 0)
  - masteryLevel = 5 的单词进入"已掌握"列表

  复习优先级:
  - masteryLevel 0-1: 每次必出
  - masteryLevel 2-3: 随机出现
  - masteryLevel 4: 偶尔出现（巩固）
  - masteryLevel 5: 不再出现在复习中
```

**数据清理逻辑：**

```typescript
// 每个单词保留 N 条录音，超出自动删除最旧的
const MAX_RECORDINGS_PER_WORD = 3;

async function saveRecording(wordId: string, audioBlob: Blob) {
  const recordings = await db.recordings.where('wordId').equals(wordId).toArray();
  
  // 如果超过上限，删除最旧的
  if (recordings.length >= MAX_RECORDINGS_PER_WORD) {
    recordings.sort((a, b) => a.createdAt - b.createdAt);
    const excess = recordings.length - MAX_RECORDINGS_PER_WORD + 1;
    for (let i = 0; i < excess; i++) {
      await db.recordings.delete(recordings[i].id);
    }
  }
  
  // 存入新的
  await db.recordings.add({
    id: crypto.randomUUID(),
    wordId,
    audioData: audioBlob,
    createdAt: Date.now()
  });
}

// 清理 N 天前的录音
async function cleanupOldRecordings(daysAgo: number) {
  const cutoff = Date.now() - daysAgo * 24 * 60 * 60 * 1000;
  const toDelete = await db.recordings
    .where('createdAt')
    .below(cutoff)
    .toArray();
  
  for (const rec of toDelete) {
    await db.recordings.delete(rec.id);
  }
  
  return toDelete.length;
}
```

## 6. 数据增长控制

### 6.1 存储预算

| 数据类型 | 单次大小 | 月增长（每天 3 课×8 词） | 年增长 |
|---------|---------|------------------------|--------|
| 单词文本 | ~0.5KB | ~3.6KB | ~44KB |
| 跟读录音 | ~100KB | ~72MB (24条/天) | ~864MB |
| 波形数据 | ~2KB | ~14KB | ~170KB |
| **合计** | | **~72MB** | **~865MB** |

> 录音是主要增长点。通过"每个单词保留 3 条录音"的策略，实际年增长降至 **~300MB** 以内。

### 6.2 存储监控

```
  监控 IndexedDB 存储用量：
  navigator.storage.estimate().then(est => {
    console.log(`已用: ${est.usage} / 配额: ${est.quota}`);
  });

  当使用量超过 80% 时，在设置页面显示警告：
  ⚠️ 存储空间即将用完，建议清理旧录音
```

## 9. 已知限制与风险

| 风险 | 影响 | 应对 |
|------|------|------|
| iOS Safari 存储限制（约 50MB 后可能被清） | 长期不用可能被系统清理 | 提示家长定期使用，重要数据备份意识 |
| TTS 声音质量因系统而异 | 发音可能不够自然 | 接受差异，后续可加云端 TTS 作为可选增强 |
| Tesseract.js 在低端设备较慢 | 拍照识别等待时间长（3-5 秒） | 显示进度条，家长等待可接受 |
| MediaRecorder 格式不统一 | iOS/Android 录音格式不同 | 仅本地播放，无需跨设备兼容 |
| iOS 需要用户手势触发 TTS | 首次使用需家长点击 | 引导页面说明 |
| PWA 无法后台运行 | 切出后可能暂停 | 数据已保存，切回即可继续 |
